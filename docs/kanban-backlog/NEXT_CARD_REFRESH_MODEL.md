# NEXT_CARD_REFRESH_MODEL

## Objetivo
Normalizar la creación recurrente de tarjetas consumibles para la vida operativa de ENGREMIAT.

## Entradas futuras
- Necesidades detectadas.
- Capacidades nuevas.
- Bloqueos.
- Nuevos proyectos humanos.
- Evidencias de validación.
- Resultados de workers.
- Notas Obsidian.
- Conversaciones OpenWebUI/Ollama.

## Salidas
- Tarjeta markdown local.
- Entrada en índice.
- Manifiesto actualizado.
- Tablero HTML actualizado.
- Prompt inline para Cline/Kanban.

## Adaptadores previstos
1. needs-to-card adapter.
2. capabilities-to-card adapter.
3. project-intake-to-card adapter.
4. obsidian-note-to-card adapter.
5. ollama-output-to-card adapter.
6. openwebui-thread-to-card adapter.

## Próxima etapa
Crear el primer adapter local: detected-input -> consumable-card markdown, en modo dry-run.
