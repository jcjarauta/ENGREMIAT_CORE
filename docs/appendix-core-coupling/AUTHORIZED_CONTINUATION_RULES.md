# AUTHORIZED CONTINUATION RULES

## Autorizacion recibida
El humano autoriza continuar hasta terminar el objetivo actual.

## Aplicacion
Esta autorizacion permite completar validaciones, cierres y preparacion local del objetivo.

## No autorizado implicitamente
- Escritura real en Obsidian sin VaultPath.
- Escritura en Kanban nativo.
- Ejecucion real OpenWebUI/OpenClaw.
- Navegador/clipboard/teclado automatico.
- Push/git remoto si no se solicita en este flujo.

## Regla
Cuando falte un dato externo imprescindible, como VaultPath, el sistema genera gate/preview y se detiene sin aplicar escritura real.
