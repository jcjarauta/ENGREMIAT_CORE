# Operational Input Draft

## Proposito operativo
Crear un artefacto reutilizable que convierta el aprendizaje del loop util anterior en una guia practica para ejecutar futuras tareas humano-maquina con menos friccion.

## Cuando usarlo
Usar este artefacto cuando se quiera preparar una tarea operativa pequena, local, autorizada por humano y con evidencia verificable.

## Inputs necesarios
- Objetivo operativo concreto.
- Output esperado.
- Limites de seguridad.
- Criterios de validacion.
- Siguiente paso recomendado.

## Pasos operativos
1. Definir la tarea concreta.
2. Capturar input editable.
3. Crear propuesta y preview.
4. Pedir autorizacion humana.
5. Generar artefacto local.
6. Validar output y rollback no destructivo.
7. Registrar evidencia y refrescar dashboard.

## Validacion
El artefacto debe ser legible, reutilizable, local-only, reversible y no debe requerir red, git write ni ejecucion externa.
