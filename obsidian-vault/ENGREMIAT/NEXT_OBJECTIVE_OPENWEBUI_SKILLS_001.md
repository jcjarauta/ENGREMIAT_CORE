# NEXT OBJECTIVE - ENGREMIAT_OPENWEBUI_SKILLS_001

## Motivo
ENGREMIAT ya tiene Knowledge Base operativa en Open WebUI y vault Obsidian listo como taller de conocimiento. Existen 3 skill-candidates iniciales suficientemente claros para iniciar una fase de Skills simples y documentales.

## Objetivo
Transformar procedimientos repetibles de ENGREMIAT en Skills reutilizables dentro de Open WebUI, empezando por capacidades de bajo riesgo basadas en lectura, resumen, validacion y preparacion de siguientes pasos.

## Skill candidates iniciales
1. [[SKILL_CANDIDATE_001_RESUMIR_ESTADO_ENGREMIAT]]
2. [[SKILL_CANDIDATE_002_LEER_ROADMAP_SIGUIENTE_PASO]]
3. [[SKILL_CANDIDATE_003_VALIDAR_BLOQUE_EVIDENCIA]]

## Orden recomendado
E01 Diseno de contrato de Skills ENGREMIAT.
E02 Crear Skill 001: resumir estado.
E03 Crear Skill 002: leer roadmap y proponer siguiente paso.
E04 Crear Skill 003: validar bloque y evidencia.
E05 Probar Skills manualmente en Open WebUI.
E06 Ajustar y documentar uso humano.
E07 Cierre readiness Skills.

## Limites
No crear agentes complejos. No ejecutar acciones externas. No sincronizar automaticamente. No automatizar desarrollo. Empezar con Skills documentales, manuales y verificables.
