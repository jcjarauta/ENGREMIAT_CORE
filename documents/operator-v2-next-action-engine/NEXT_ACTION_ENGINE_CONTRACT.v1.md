# NEXT ACTION ENGINE CONTRACT v1

## Proposito
Convertir `next_action` en una decision operativa visible, explicable y accionable dentro del Operador V2.

## Principios
- No construir V1 a ciegas: cada siguiente paso debe salir del estado vivo V2.
- Separar diagnostico, recomendacion, riesgo y accion.
- Mantener modo read-only hasta validar pantalla y contrato.
- No ejecutar acciones sensibles desde el motor.

## Pantalla objetivo
OPERADOR V2 / SIGUIENTE ACCION

Debe mostrar: objetivo actual, estado/readiness, NEXT, recomendacion, por que, riesgo, acciones permitidas, acciones bloqueadas y comando siguiente.
