# ENGREMIAT - Dossier base para subvenciones

## 1. Resumen ejecutivo
Engremiat es una infraestructura local-first para transformar conocimiento disperso en memoria viva, documentación útil y trabajo trazable. El sistema organiza proyectos mediante módulos, rutas, reports, diario de desarrollo, dossiers reutilizables y futura colaboración con workers locales.

Su objetivo es reducir la fricción humana en la coordinación, documentación, continuidad y justificación de proyectos comunitarios, cooperativos, tecnológicos y de economía circular. La propuesta permite generar evidencias y documentos desde el propio estado operativo del sistema.

## 2. Problema detectado
Muchos proyectos con vocación social, comunitaria o innovadora no fracasan por falta de ideas, sino por pérdida de continuidad, falta de documentación, dispersión de tareas, dificultad para justificar impacto y dependencia excesiva de memoria humana no estructurada.

Esta situación provoca:
- Dificultad para retomar el trabajo entre sesiones.
- Pérdida de decisiones, aprendizajes y evidencias.
- Problemas para preparar subvenciones o memorias justificativas.
- Baja trazabilidad entre necesidades, acciones, recursos y resultados.
- Dependencia de personas concretas para sostener el conocimiento del proyecto.

## 3. Solución propuesta
Engremiat propone una base operativa local-first que organiza el proyecto en módulos, genera memoria viva, produce documentación desde evidencias reales y prepara la incorporación progresiva de workers locales para apoyar redacción, análisis y ejecución asistida.

La solución se compone de:
- Inicio humano estable para navegar el sistema.
- Módulos de proyecto y biblioteca reutilizable.
- Workspace y tarjetas por módulo.
- Árbol de navegación y rutas recomendadas.
- Estado técnico, status externo y logs.
- Centro de memoria y documentación.
- Dossiers para subvenciones, partners, clientes y contexto worker.
- Exportación documental local-first con fallback Markdown y futura conexión Pandoc.

## 4. Innovación

- Documentación automática sin depender inicialmente de servicios externos.

## 8. Metodología de trabajo
La metodología se basa en ciclos cortos de construcción, validación humana y generación de evidencias. Cada mejora del sistema queda registrada mediante reports, logs y documentación generada.

Fases metodológicas:
- Detectar necesidad o fricción operativa.
- Crear o ajustar módulo/pantalla/script.
- Validar comportamiento en terminal.
- Registrar report y estado.
- Actualizar memoria viva.
- Generar documentación reutilizable.
- Revisar con humano antes de ampliar automatización.

## 15. Riesgos y mitigación
### Riesgo: exceso de complejidad técnica
Mitigación: desarrollo por fases, una frontera técnica por bloque y validación humana.

### Riesgo: dependencia de herramientas externas
Mitigación: local-first, fallback Markdown y conexión opcional a herramientas open source.

### Riesgo: documentación demasiado técnica
Mitigación: separar memoria técnica, resumen humano y dossiers por destinatario.

### Riesgo: automatización prematura
Mitigación: worker desactivado por defecto y activación solo con autorización explícita.

## 16. Sostenibilidad
Engremiat está diseñado para crecer de forma modular. Cada nueva función puede documentarse, reutilizarse y conectarse a otras capas sin depender de una única herramienta. El enfoque local-first permite mantener control de datos, reducir costes iniciales y evolucionar hacia automatización cuando el sistema esté preparado.

## 17. Escalabilidad
La arquitectura permite pasar de un proyecto individual a múltiples proyectos, módulos, bibliotecas reutilizables, plantillas documentales y workers especializados. En fases posteriores, el sistema puede ampliarse a Telegram, dashboards, exportaciones avanzadas, formularios guiados y gestión documental colaborativa.
