# KANBAN_BACKLOG_OPERATION_MANUAL

## Uso

1. Completar las 25 tarjetas en 03_CARDS/cards.
2. Revisar SYSTEM_NEEDS_INDEX.
3. Ejecutar autoanalisis o refresh controlado.
4. Revisar REFRESH_PROPOSAL.
5. Decidir prioridades humanas.
6. Generar bundle para Ollama/OpenWebUI.

No crear nueva boveda para estas tarjetas. Viven dentro de KANBAN_BACKLOG como SYSTEM_NEEDS.
