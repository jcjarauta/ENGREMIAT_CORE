# CAPTURE TO SKILL FLOW

## Flujo

1. Pegar captura útil en [[../07_CAPTURAS/CAPTURE_INBOX]].
2. Convertir la captura a una fila en [[SKILL_CANDIDATE_LOG]].
3. Probar si el procedimiento se repite.
4. Si reduce fricción humana, marcar como OFFICIAL_CANDIDATE.
5. Solo después preparar skill formal.

## Criterios GO

- La respuesta o procedimiento se ha usado más de una vez.
- El humano entiende cuándo invocarlo.
- Produce una salida estable.
- Reduce búsqueda, memoria dispersa o navegación manual.

## Criterios NO_GO

- Depende de contexto demasiado ambiguo.
- Solo funcionó una vez.
- Mezcla demasiados objetivos.
- Requiere automatización pesada antes de estar claro.
