# ENTITY_DICTIONARY

| Entidad | Definicion | Uso |
|---|---|---|
| Objective | Objetivo de desarrollo cerrado o activo | Agrupa etapas, bloques, pasos y evidencias |
| Stage | Etapa dentro de un objetivo | Ordena avances grandes |
| Block | Bloque funcional dentro de una etapa | Agrupa pasos tecnicos |
| Step | Unidad minima ejecutable y validable | Se valida con evidencia |
| Artifact | Archivo producido o actualizado | Debe tener ruta y hash cuando aplique |
| Error | Fallo observado | Guarda causa, reparacion y prevencion |
| Decision | Eleccion humana o tecnica relevante | Explica por que se toma un camino |
| Gate | Autorizacion humana requerida | Bloquea acciones sensibles |
| Worker | Agente o asistente que propone trabajo | No decide por el humano |
| Commit | Cierre Git local o remoto cuando aplique | Se asocia a objetivo o etapa |
| Evidence | Prueba local de que algo ocurrio | Reporte, manifest, checksum, smoke o dashboard |
