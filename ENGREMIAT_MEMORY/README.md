# ENGREMIAT_MEMORY

Memoria visual local del desarrollo de ENGREMIAT.

## Proposito
Crear una capa humana, navegable y versionable para entender objetivos, etapas, bloques, pasos, decisiones, errores, gates, workers, evidencias y artefactos.

## Relacion con ENGREMIAT_CORE
ENGREMIAT_CORE ejecuta, valida y produce evidencias. ENGREMIAT_MEMORY explica, enlaza y hace navegable el desarrollo para el humano y para futuras IAs.

## Regla principal
La memoria visual no sustituye los reportes JSON ni los manifests: los resume, enlaza y ordena para uso humano.

## Carpetas canonicas
- objectives/: objetivos de desarrollo.
- stages/: etapas.
- blocks/: bloques.
- steps/: pasos.
- errors/: errores recurrentes.
- decisions/: decisiones humanas o tecnicas.
- artifacts/: artefactos destacados.
- evidence/: evidencias verificables.
- gates/: gates humanos.
- workers/: workers asistidos.
- maps/: mapas visuales.
- templates/: plantillas reutilizables.
- logseq/: workspace visual compatible con Logseq.
- graph/: exportacion tecnica futura.

## Seguridad
LOCAL_FIRST, EVIDENCE_FIRST, FAIL_CLOSED y HUMAN_GATE_BEFORE_APPLY.
