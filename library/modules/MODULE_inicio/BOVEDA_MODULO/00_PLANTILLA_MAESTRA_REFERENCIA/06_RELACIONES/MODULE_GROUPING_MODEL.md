# MODULE_GROUPING_MODEL

## Proposito
Definir como se agrupa manualmente un modulo para entender su trabajo, dependencias y carga humana.

## Enlaces esperados
- Home modulo: [[MODULE_HOME]]
- Objetivo modulo: [[MODULE_OBJECTIVE_MODEL]]
- Tareas: [[MODULE_TASK_MODEL]]
- Evidencias: [[MODULE_EVIDENCE_MODEL]]
- Gates: [[MODULE_GATE_MODEL]]
- Resultados: [[MODULE_RESULT_MODEL]]

## Grupos internos del modulo
1. Alcance del modulo.
2. Tareas humanas.
3. Tareas asistidas.
4. Tareas automatizables.
5. Evidencias necesarias.
6. Gates o autorizaciones.
7. Dependencias con otros modulos.
8. Bloqueos y riesgos.
9. Salidas reutilizables.

## Regla
Un modulo no debe ser solo una carpeta: debe ser una unidad de trabajo agrupable, revisable y conectable al grafo del proyecto.
