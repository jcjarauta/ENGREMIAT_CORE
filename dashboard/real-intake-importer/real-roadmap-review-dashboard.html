ENGREMIAT · Real Roadmap Review

E02 PASS · Dashboard de revision creado

Proyecto: Primer intake real para construir roadmap automático en ENGREMIAT

Intake base

Necesidad: Necesitamos una forma sencilla de introducir una idea humana de proyecto y convertirla en una estructura ordenada de desarrollo, sin depender de navegador, memoria dispersa ni prompts largos repetidos.

Resultado deseado: Obtener un intake exportable en JSON y Markdown que pueda alimentar una propuesta de roadmap revisable por el humano antes de preparar cualquier worker o automatización.

Restricciones:
Solo local.
Sin backend.
Sin red.
Sin worker real.
Sin Git.
Sin ejecución automática.
Exportación manual.
Debe ser comprensible para humano y útil para generar roadmap.

Criterios de exito:
La UI abre correctamente.
El formulario permite rellenar todos los campos.
El botón Generar intake produce JSON válido.
El JSON contiene safety.local_only=true.
El JSON contiene worker_execution=false.
El JSON contiene execute_now=false.
El Markdown descargado resume correctamente el proyecto.
El resultado permite pasar a una etapa E02 de roadmap preview.

Roadmap propuesto

E01 · Normalizar intake real
Guardar el JSON exportado como intake aceptado y canónico.
gate: PASS
E02 · Roadmap preview
Convertir el intake en etapas revisables antes de worker.
gate: REVIEW
E03 · Gate humano
Aprobar, editar, rechazar o aplazar el roadmap.
gate: REQUIRED
E04 · Worker packet preview
Preparar paquete para worker sin ejecución real.
gate: REQUIRED
E05 · Dry-run controlado
Simular la ejecución y mantener NO_GO sin autorización explícita.
gate: REQUIRED

Decision humana pendiente

Botones informativos: esta UI no ejecuta decisiones. El recibo queda pendiente para operador local.

Seguridad
